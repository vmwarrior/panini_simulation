You have access to a python environment located in .venv
Your job is to write code to estimate how much it would cost to completely fill a 2026 Fifa World Cup Panini album. There are 980 stickers in the album. The stickers come in packets of 7, and you can buy bundles of 25 packets for $50. You can run a simulation to estimate how many packets you need to buy, assuming that the stickers are placed in a packet randomly. There are no repeated stickers in each packet. For now assume that you cannot trade cards with other users, so the only way to fill the album is by buying packets. Can you designa a MonteCarlo simulation for this?

Now simulate how this estimate changes when trading of stickers is allowed. The way this should work is every time you buy a new 25-packet bundle, you can randomly simulate that each missing sticker has a probability p = D / (D+N) of being traded for one of the repeated stickers, where D = number of repeated stickers, and N=980. Every time you trade a sticker, you need to make sure that you decrease the number of duplicate stickers you are holding. Once trading is not succesful, you need to buy a new 25-packet bundle.

How would it change the results if you wanted to collect 10 albums instead of just 1?
